
# Reconciliation Reports Feature Implementation Plan

## Overview
Add a new "Reconciliation Reports" feature to the Business section that allows users to generate, view, and export comprehensive tax reconciliation reports from existing Income, Expenses, and Tax data.

## Architecture Summary

### Database Schema
A new `reconciliation_reports` table will store generated reports with metadata including:
- Report ID, business ID, generated by user ID
- Report type (tax_summary, adjustments, evidence_exceptions)
- Period range and tax type
- Status (draft, approved) for owner approval workflow
- Export file URLs (PDF/Excel stored in Supabase Storage)
- Created/approved timestamps

### Access Control
Row-Level Security policies will enforce:
- **SME Owners**: Full CRUD for their business reports
- **Accountants**: View/Generate for assigned businesses only
- **Admins**: Full access to all reports

### UI Components
New components following existing patterns:
- `ReconciliationReports.tsx` - Main page with report list and builder
- `ReconciliationReportBuilder.tsx` - Filters and generation form
- `TaxReconciliationSummary.tsx` - Summary report display
- `AdjustmentsSchedule.tsx` - Adjustments breakdown
- `EvidenceExceptionsReport.tsx` - Missing docs and anomalies

## Implementation Steps

### Step 1: Database Migration

Create the `reconciliation_reports` table and a new storage bucket for report files.

**Table Schema:**
```sql
CREATE TABLE public.reconciliation_reports (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  business_id UUID NOT NULL REFERENCES businesses(id) ON DELETE CASCADE,
  generated_by UUID NOT NULL REFERENCES profiles(id),
  report_type TEXT NOT NULL, -- 'tax_summary', 'adjustments', 'evidence_exceptions'
  tax_type TEXT NOT NULL, -- 'income', 'presumptive', 'paye', 'vat'
  period_start DATE NOT NULL,
  period_end DATE NOT NULL,
  report_data JSONB NOT NULL DEFAULT '{}'::jsonb,
  status TEXT NOT NULL DEFAULT 'draft', -- 'draft', 'approved'
  approved_by UUID REFERENCES profiles(id),
  approved_at TIMESTAMPTZ,
  pdf_url TEXT,
  excel_url TEXT,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

**RLS Policies:**
- Owners can manage reports for their businesses
- Accountants can view/insert reports for assigned businesses
- Admins have full access
- Only owners can update status to 'approved'

**Storage Bucket:**
- Create `reconciliation-reports` private bucket for PDF/Excel exports

### Step 2: Create Calculation Utilities

Create `src/lib/reconciliationCalculations.ts` with functions to:

1. **calculateTaxReconciliationSummary()**
   - Fetch income and expenses for period
   - Calculate: Total Income, Total Expenses, Net Profit
   - Determine adjustments (add-backs like depreciation, non-deductible expenses)
   - Calculate Taxable Profit using existing tax calculation logic
   - Compute Estimated Tax Liability (labeled as estimate)

2. **generateAdjustmentsSchedule()**
   - Identify expense categories that are typically add-backs
   - Link adjustments to underlying entries
   - Include evidence status for each adjustment

3. **generateEvidenceExceptionsReport()**
   - Check for income/expenses without document uploads
   - Flag entries with amounts above threshold (anomaly detection)
   - Identify entries edited after month lock
   - Collect validation warnings from existing data

### Step 4: Export Functionality

Extend `src/lib/exportImport.ts` with reconciliation-specific exports:

1. **exportReconciliationToPDF()**

   - Summary totals section

### Step 5: Add Navigation

Update `src/pages/businesses/BusinessDetail.tsx`:
- Add "Reports" button alongside Income/Expenses buttons in header

Each log includes: business_id, report_id (in details), user_id, timestamp

### Step 7: Translations

Add keys to both `en.ts` and `lg.ts`:

## Technical Details

### Report Data Structure (JSONB)

```typescript
interface TaxReconciliationData {
  totalIncome: number;
  totalExpenses: number;
  netProfit: number;
  adjustments: {
    addBacks: { category: string; amount: number; reason: string }[];
    deductions: { category: string; amount: number; reason: string }[];
  };
  taxableProfit: number;
  estimatedTax: number;
  incomeBreakdown: { source: string; amount: number }[];
  expenseBreakdown: { category: string; amount: number }[];
}

interface AdjustmentsData {
  items: {
    id: string;
    category: string;
    reason: string;
    amount: number;
    entryId: string;
    entryType: 'income' | 'expense';
    hasEvidence: boolean;
  }[];
}

interface EvidenceExceptionsData {
  missingReceipts: { id: string; type: string; amount: number; date: string }[];
  largeEntries: { id: string; type: string; amount: number; threshold: number }[];
  editedAfterLock: { id: string; type: string; editedAt: string }[];
  validationWarnings: { id: string; message: string }[];
}
```

### Anomaly Detection Thresholds
- Large entry threshold: Entries above 3x the average for that category
- Unusual patterns: Month-over-month variance above 200%

### Period Handling
- Monthly: `YYYY-MM` format (e.g., "2026-02")
- Quarterly: Q1 (Jan-Mar), Q2 (Apr-Jun), Q3 (Jul-Sep), Q4 (Oct-Dec)
- Annual: Full year matching business tax type

2. **Owner-Only Approval**: Only business owners can mark reports as "Approved"
3. **Storage Security**: Report files in private bucket with signed URLs
4. **Audit Logging**: All actions logged for compliance
